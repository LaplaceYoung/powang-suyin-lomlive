# Project Brief

## Current Problem

- 目标用户：需要进行实时语音通话身份可信判断的用户、平台和安全系统。
- 使用场景：VoIP 通话、远程身份核验、语音诈骗防护、实时语音安全监测。
- 核心痛点：语音合成和录音重放降低了语音诈骗门槛，单纯依赖用户识别难以稳定判断语音来源真实性。
- 现有方案缺口：传统语音活体检测常依赖特定传感器、位置或姿势约束，实际通话场景部署成本较高。

## Plan

1. 核心系统：在 VoIP 应用与计算机音频流之间构建实时检测中间层。
2. 核心 AI 技术：Lombard 效应建模、双输入语音对比、dif-SE-Res2Net 差分增强网络。
3. 工程闭环：噪声注入、音频采集、特征提取、活体判断、检测反馈。
4. 前端展示方式：实时通话检测流程、合成语音和重放攻击对比测试、模型性能指标展示。

## Success Criteria

- 指标 1：活体检测准确率、误报率、漏报率。
- 指标 2：不同噪声、设备和通话环境下的鲁棒性。
- 指标 3：实时检测延迟和资源占用。
- 落地证据：项目研究报告、系统功能测试、性能测试、作品视频或可运行入口。

## Risks

- 技术风险：Lombard 效应在不同用户和设备上的稳定性需要用实验数据支撑。
- 数据风险：真实通话、合成语音、重放攻击样本覆盖度会影响说服力。
- 查新风险：需要证明实时通话场景下的 Lombard 活体检测组合具有新颖性。
- 前端展示风险：实时音频链路、虚拟声卡和 VoIP 软件兼容性需要提前验证。
